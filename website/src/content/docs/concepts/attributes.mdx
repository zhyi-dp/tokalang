title: Attribute Tokens
description: Discrete tokens for mutability and nullability in Toka.
---

Toka replaces verbose keywords like `mutable` or `optional` with a system of concise attribute tokens. These tokens are discrete and can be combined to define the exact behavior of souls and handles.

## Discrete Dimensions

### 1. Soul Mutability (`#` suffix)
The `#` token applied as a suffix to a variable or field name indicates that the **Soul** (the data itself) is mutable.

```toka
auto x# = 10
x = 20 // OK
```

### 2. Entity Nullability (`?` suffix)
The `?` token applied as a suffix to a **Type** indicates that the entity can be `none`.

```toka
auto x: i32? = none
if x == none {
    println("No value")
}
```

### 3. Pointer Nullability (`nul` prefix)
The `nul` keyword applied as a prefix to a pointer name indicates that the **Handle** (the pointer itself) can be `nullptr`.

```toka
auto nul ^p: User = nullptr
if ^p == nullptr {
    println("No pointer")
}
```

### 4. Handle Rebindability (`^#`, `~#`)
The `#` token applied after a pointer sigil indicates that the **Handle** itself is mutable (can be reassigned to a different address).

```toka
auto ^#p = new i32(10)
^p = ^q // OK: Rebinding the handle
```

## The "Chemical Formula" Approach

Attributes can be stacked like a chemical formula to define complex behaviors. 

### Example: A Rebindable Nullable Pointer to a Mutable Nullable Entity

```toka
auto nul ^#node#: Node? = nullptr
```

Breakdown:
1. `nul`: The pointer can be `nullptr`.
2. `^#`: The pointer is unique and can be rebound (`^node = ^other`).
3. `node#`: The data pointed to is mutable (`node.value = 10`).
4. `Node?`: The entity itself can be `none`.

## Summary Table

| Token | Position | Dimension | Effect |
| :--- | :--- | :--- | :--- |
| **`#`** | Suffix to Name | Soul Mutability | Allows modifying the data |
| **`?`** | Suffix to Type | Entity Nullability | Allows the value to be `none` |
| **`nul`** | Prefix to Name | Pointer Nullability | Allows the pointer to be `nullptr` |
| **`^#`** | Prefix to Name | Handle Mutability | Allows rebinding the pointer |
