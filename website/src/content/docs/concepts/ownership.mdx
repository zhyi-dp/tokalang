title: Resource Safety & Ownership
description: Ownership, RAII, and manual memory management in Toka.
---

Toka ensures resource safety through a combination of strict ownership rules and the **RAII (Resource Acquisition Is Initialization)** pattern.

## Ownership Rules

Ownership in Toka is defined by **exclusive unique pointers (`^`)**.

1. **Unique Owner**: Each resource (memory, file handle, socket) has exactly one unique owner at any given time.
2. **Move by Default**: Assigning an owned pointer moves the ownership, making the original handle invalid.
3. **Explicit Transfer**: Use the `cede` keyword to explicitly transfer ownership, especially when passing data into closures or multi-threaded tasks.

```toka
fn take_ownership(^u: User) {
    // u will be dropped here
}

pub fn main() -> i32 {
    auto ^u = new User(name = "Bob")
    take_ownership(cede u) // Explicit move
    // u cannot be used here
    return 0
}
```

## RAII and the @encap Trait

Toka uses the `impl Type@encap` block to define how resources should be cleaned up. This is equivalent to a "destructor" or "drop" function.

```toka
shape FileHandle (
    fd: i32
)

impl FileHandle@encap {
    // The compiler automatically synthesizes a drop mechanism
    // that calls this logic when a FileHandle goes out of scope.
    pub fn drop(self) {
        if self.fd > 0 {
            libc_close(self.fd)
            println("File handle closed.")
        }
    }
}
```

## Borrowing (References)

To use data without taking ownership, use **borrowing (`&`)**.

1. **Safety**: Borrows must never outlive their owners.
2. **Immutability by Default**: A borrow `&T` is read-only.
3. **Mutable Borrow**: Use `&T#` to allow modifying the pointed-to entity.

```toka
pub fn process_data(&u: User) {
    // Read-only access
}

pub fn update_user(&u#: User) {
    // Mutable access to the user soul
    u.name = "New Name"
}
```

## Moving and Return Values

When a function returns an owned object, it is automatically moved to the caller. No expensive copy or explicit drop occurs.

```toka
fn create_user() -> ^User {
    return new User(name = "Charlie") // Ownership moved to caller
}
```

This model provides the safety of garbage-collected languages with the predictable performance of manual memory management.
