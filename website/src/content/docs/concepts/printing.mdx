Toka features an incredibly modern, secure, and high-performance string formatting and output system. We have completely rejected traditional C-style `%d`/`%s` specifiers in favor of an intuitive `{}` placeholder syntax backed by compile-time type-safety checks.

## Basic Printing (`println`)

To use printing in Toka, you must first import `std/io::println` from the standard library:

```toka
import std/io::println

pub fn main() -> i32 {
    println("Hello, Toka!")
    return 0
}
```

## Placeholder Interpolation

Use `{}` as the basic placeholder. Any entity type implementing the `@ToString` trait can be interpolated:

```toka
auto name = "Alice"
auto age = 18
println("Name: {}, Age: {}", name, age)
```

### Zero-Copy Optimization
For maximum performance, when the interpolated value is already a `String` or `view_str`, the `println` macro completely bypasses extra copying or heap allocations, performing a direct zero-copy write under the hood.

## Fine-Grained Mini Formatting Language

To satisfy advanced printing requirements such as base conversion, leading zero-padding, and float precision control, Toka supports a **fine-grained formatting mini-language** with the syntax `{:command}`.

### 1. Leading Zero-Padding (`{:0N}`)
Controls the minimum width of the output and pads with leading zeros if necessary:

```toka
auto val = 42
println("Zero-padded: {:04}", val) // Output: 0042
```

### 2. Base Conversion (`{:x}` / `{:X}` / `{:o}`)
Formats integers into lowercase/uppercase hexadecimal or octal:

```toka
auto hex_val: usize = 0xABCD
println("Hex lowercase: 0x{:x}", hex_val) // Output: 0xabcd
println("Hex uppercase: 0x{:X}", hex_val) // Output: 0xABCD
```

### 3. Floating-Point Precision (`{:.N}`)
Controls the number of decimal places for floating-point values (`f32`/`f64`). **Note: You do not need redundant type characters (like `f`) inside the placeholders; the compiler resolves it based on the data type.**

```toka
auto pi = 3.1415926
println("Two decimals: {:.2}", pi) // Output: 3.14
println("Four decimals: {:.4}", pi) // Output: 3.1416
```

## Underlying Traits: `@ToString` and `@ToFormat`

This modern formatting system is driven by underlying trait contracts in the standard library rather than rigid compiler hardcoding:

| Placeholder | Trait Bound | Required Method | Compilation Behavior |
| :--- | :--- | :--- | :--- |
| `{}` | `@ToString` | `to_string(self) -> String` | Sema and CodeGen automatically resolve the implementation and generate a conversion call. |
| `{:command}` | `@ToFormat` | `to_string_fmt(self, fmt: view_str) -> String` | The compiler extracts the command after `:` (e.g., `"04"`) at compile time, packages it as a `view_str`, and injects it as the second argument. |

### Custom Shape Formatting
Thanks to the loose coupling of `@ToFormat`, you can implement custom format specifiers for any user-defined Shape:

```toka
shape Point(x: i32, y: i32)

impl Point {
    pub fn to_string_fmt(self, fmt: view_str) -> String {
        if fmt == s"json" {
            // Custom JSON serialization
            return String::format("{{\"x\": {}, \"y\": {}}}", self.x, self.y)
        }
        // Fallback to normal coordinate format
        return String::format("({}, {})", self.x, self.y)
    }
}
```

Which can be cleanly used like this:
```toka
auto p = Point(x = 10, y = 20)
println("JSON Format: {:json}", p) // Output: {"x": 10, "y": 20}
```
