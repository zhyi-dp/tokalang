Toka's memory safety model is built into its **Pointer Morphology**. Instead of generic smart pointer types like `Box<T>` or `Rc<T>`, Toka uses specific sigils (symbols) to define the relationship between a pointer and the data it points to.

## Pointer Types

Toka provides four primary pointer types, each with its own ownership and lifecycle semantics.

### 1. Unique Pointer (`^`)
The `^` token denotes **exclusive ownership**. There can be only one unique pointer to a given piece of data at any time.
- **RAII**: When a `^` pointer goes out of scope, the memory is automatically freed.
- **Move Semantics**: Ownership is moved by default. Use `cede` to explicitly transfer it.

```toka
pub fn delete_user(^u: User) {
    // u will be dropped at the end of this function
}

pub fn main() -> i32 {
    auto ^u = new User(name = "Alice")
    delete_user(cede u) // 'cede' transfers ownership to the function
    // u is now invalid here
    return 0
}
```

### 2. Shared Pointer (`~`)
The `~` token denotes **shared ownership** with reference counting.
- **Automatic**: Reference counts are handled by the compiler.
- **Safe**: Data remains alive as long as at least one `~` pointer exists.

```toka
shape Node (
    data: i32,
    nul ~next: Node // Shared, nullable pointer
)
```

### 3. Borrow / Reference (`&`)
The `&` token is used for **borrowing**. It provides a temporary, non-owning view.
- **Safety**: The borrow checker ensures that a borrow does not outlive its owner.
- **Writability**: Use `&p#` for a mutable borrow (where the entity is mutable).

```toka
pub fn print_name(&u: User) {
    println("User name: {}", u.name)
}
```

### 4. Raw Pointer (`*`)
The `*` token represents a **raw, C-style pointer**.
- **Unsafe**: Used for FFI or low-level memory operations.

## Mutability and Rebinding

Toka distinguishes between the **Writability** of the pointed-to data and the **Rebindability** of the pointer itself.

| Code | Meaning |
| :--- | :--- |
| `auto ^p` | Read-only unique pointer. |
| `auto ^p#` | Unique pointer to a **mutable** entity. |
| `auto &#p` | A **rebindable** reference. |
| `auto nul ^#node#: Node?` | A nullable (`nul`), rebindable (`#` after ^) unique pointer to a mutable (`#` after node) and nullable (`?`) Node. |

## The "Hat" Principle

In Toka, a bare variable name like `p` represents the **Soul** (the value). Accessing `p` automatically dereferences it. To operate on the **Handle** (the pointer address), you "put on a hat" using the sigils.

- `p = value` (Modifies the data)
- `^p = ^q` (Rebinds the unique pointer)

This makes the distinction between value-operations and address-operations visually explicit.
