For a fast and hassle-free setup, we provide pre-compiled binaries that **natively support Windows, macOS, and Linux**. Thanks to the internally integrated linker (LLD), you no longer need to configure any external C++ build environment to compile and link your executables.

## Quick Install (macOS & Linux)

If you are using macOS or Linux, you can install Toka and all its associated tools (the compiler, package manager, and standard library) by running the following command in your terminal:

```bash
curl -fsSL https://tokalang.dev/install.sh | bash
```

This script will automatically detect your OS and CPU architecture, download the latest stable release to `~/.toka/`, and append the executables to your `PATH` and `TOKA_LIB` variables.

## Windows Installation

For Windows users, Toka provides native `.exe` pre-compiled binaries:

1. Go to the [GitHub Releases](https://github.com/zhyi-dp/tokalang/releases) page.
2. Download the latest archive with the `Windows` suffix.
3. Extract the archive to a directory of your choice (e.g., `C:\tokalang`).
4. Add the `bin` folder inside the extracted directory to your system's `PATH` environment variable.
5. Add the `lib` folder inside the extracted directory to a new system environment variable named `TOKA_LIB`.

> 💡 **Tip:** The Windows version has been deeply refactored and adapted to the underlying Win32 API, perfectly supporting a native development and debugging experience!

## Quick Start (Hello World)

Once installed, you can copy the following code to instantly experience Toka:

```bash
# 1. Quickly write a classic Hello World file
echo 'import std/io::{println}

fn main() {
    println("Hello, Toka World!")
}' > hello.tk

# 2. Compile and run automatically in one click!
toka run hello.tk
```

## Build From Source

If you wish to contribute to the compiler's C++ core, you can easily build the toolchain from source.

### Prerequisites

- **LLVM 20**: The Toka framework is heavily coupled with LLVM 20.
- **CMake**: Version 3.15 or higher.
- A C++ build toolchain for your OS (e.g., MSVC, GCC, Clang).

### Compiling

Toka provides a convenient automated build script `rebuild.sh` that compiles both the core compiler `tokac` and the package manager `toka`.

1. Clone the repository:
   ```bash
   git clone https://github.com/zhyi-dp/tokalang.git
   cd tokalang
   ```

2. Configure the CMake project (for the first time):
   ```bash
   mkdir build && cd build
   cmake ..
   cd ..
   ```

3. Run the complete rebuild script from the root directory:
   ```bash
   ./rebuild.sh
   ```

After compilation, both binaries will be available in the `build/bin` directory.

## Environment Configuration

To use Toka seamlessly from any directory, add these configurations to your `.zshrc` or `.bashrc`:

```bash
# Add the compiler and build tool to your PATH
export PATH="/path/to/tokalang/build/bin:$PATH"

# Essential: Tell the compiler and package manager where the standard library is
export TOKA_LIB="/path/to/tokalang/lib"

# Third-Party Libraries (New): Define global dependency search paths
# You can use ":" to split multiple paths. Drop your downloaded or personal
# library folders here, and you can seamlessly `import mylib::{...}` in your code!
export TOKA_PATH="/path/to/your/custom/libs:/another/lib/path"

# (Optional) Specify the Clang compiler for linking.
export TOKA_CLANG="/usr/local/opt/llvm@20/bin/clang"
```

> [!IMPORTANT]
> Make sure to replace `/path/to/tokalang` with the actual absolute path to your cloned repository.

## Package Manager Workflow

Once the environment variables are set up, you can use the native `toka` command from any directory!

### 1. Initialize a New Project

Use the `new` command to quickly scaffold a project with a standardized structure:

```bash
toka new my_app
cd my_app
```

This generates:
- `src/main.tk`: The project entry file.
- `toka.mod`: Toka's declarative package manifest.

### 2. Importing Files & Dependencies

Toka provides an incredibly intuitive and straightforward `import` module system:

**Importing Local Files (Relative & Absolute)**:
Toka natively supports writing paths directly in code (Note: no quotes needed!):
```toka
// Relative path import for utils.tk in current directory
import ./utils::{foo, bar}
import ../lib/network::{get}

// Absolute path import
import /opt/shared/mylib::{baz}
```

**Importing Package Manager Dependencies**:
If you installed a 3rd-party package via `toka add pkg.tokalang.dev/user/ansi`, it gets registered. You can then import it naturally:
```toka
import ansi::{...}
```

> **Tip:** When resolving bare names like `ansi`, the compiler searches sequentially through the standard library (`TOKA_LIB`) and your configured global paths (`TOKA_PATH`)..

### 3. Build and Run 

For continuous testing, simply orchestrate standard builds via:

```bash
toka run
```

This ensures required packages are downloaded, compile your source files efficiently, and instantly executes your application. Use `toka build` instead if you simply wish to compile a binary.
