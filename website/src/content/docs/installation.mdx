For a fast and hassle-free setup, we provide an automated installation script that downloads pre-compiled binaries for macOS and Linux.

## Quick Install (Recommended)

To install Toka and all its associated tools (the compiler, package manager, and standard library), run the following command in your terminal:

```bash
curl -fsSL https://tokalang.dev/install.sh | bash
```

This script will automatically detect your OS and CPU architecture, download the latest stable release to `~/.toka/`, and append the executables to your `PATH` and `TOKA_LIB` variables.

> **Important Requirement:** Toka currently relies on **LLVM 20** (specifically `clang`) as its backend for code generation and linking. Please ensure LLVM 20 is installed on your system and available in your `PATH` (or specified via `TOKA_CLANG`) before using the toolchain.

## Quick Start (Hello World)

Once installed, you can copy the following code to instantly experience Toka:

```bash
# 1. Quickly write a classic Hello World file
echo 'import std/io::{println}

fn main() {
    println("Hello, Toka World!")
}' > hello.tk

# 2. Compile and run automatically in one click!
toka run hello.tk
```

## Build From Source

If you wish to contribute to the compiler, or run Toka on an officially unsupported platform, you can easily build the toolchain from source.

### Prerequisites

- **LLVM 20**: The Toka framework is heavily coupled with LLVM 20.
- **Clang 20**: Required for linking the generated object files.
- **CMake**: Version 3.15 or higher.

### Compiling

Toka provides a convenient automated build script `rebuild.sh` that compiles both the core compiler `tokac` and the package manager `toka`.

1. Clone the repository:
   ```bash
   git clone https://github.com/zhyi-dp/tokalang.git
   cd tokalang
   ```

2. Configure the CMake project (for the first time):
   ```bash
   mkdir build && cd build
   cmake ..
   cd ..
   ```

3. Run the complete rebuild script from the root directory:
   ```bash
   ./rebuild.sh
   ```

After compilation, both binaries will be available in the `build/bin` directory.

## Environment Configuration

To use Toka seamlessly from any directory, add these configurations to your `.zshrc` or `.bashrc`:

```bash
# Add the compiler and build tool to your PATH
export PATH="/path/to/tokalang/build/bin:$PATH"

# Critical: Set the exact path to the Toka standard library
export TOKA_LIB="/path/to/tokalang/lib"

# (Optional) Specify the Clang compiler to use for linking. Defaults to /usr/local/opt/llvm@20/bin/clang
export TOKA_CLANG="/usr/local/opt/llvm@20/bin/clang"
```

> [!IMPORTANT]
> Be sure to replace `/path/to/tokalang` with the actual absolute path to your cloned repository.

## Developing with the Package Manager

Once your environment variables are set, you can use the native `toka` CLI directly.

### 1. Initialize a New Project

Use the `new` command to instantly scaffold a standardized project:

```bash
toka new my_app
cd my_app
```

This generates:
- `src/main.tk`: The main entry point.
- `toka.mod`: Toka's declarative build and dependency manifest.

### 2. Add Third-Party Requirements

To fetch third-party libraries seamlessly, use the `add` command which registers dependencies in your local `.tokapkg` directory:

```bash
toka add pkg.tokalang.dev/user/ansi
```

Then you can import them directly in your code via `import ansi::{...}`.

### 3. Build and Run 

For continuous testing, simply orchestrate standard builds via:

```bash
toka run
```

This ensures required packages are downloaded, compile your source files efficiently, and instantly executes your application. Use `toka build` instead if you simply wish to compile a binary.
