Toka ships with a powerful, zero-configuration command-line manager designed to give you a frictionless developer experience. Instead of interacting with the raw `tokac` compiler, you will primarily use the `toka` command to manage your projects.

## Project Scaffolding

Starting a new project in Toka is instantaneous and automatically enforces best practices.

### 1. `toka new <name>`
Creates a new **Executable Application** project. It automatically generates a `src/main.tk` entry point and a `Project.tk` file configured for building an executable binary.

```bash
toka new my_app
```

### 2. `toka new <name> --lib`
Creates a new **Third-Party Library** project. It automatically skips `main.tk` and scaffolds the exact `lib/<name>/mod.tk` structure recommended by the official [Project Structure Guidelines](/project-structure).

```bash
toka new my_lib --lib
```

## Development & Build

### 1. `toka run`
Instantly compiles and executes your project. It looks for your executable definition in `Project.tk` and runs it.

### 2. `toka run <file.tk>`
Operates as a **Single File Script Runner**. Use this if you just want to quickly test a single `.tk` file without setting up a full project directory.

### 3. `toka build`
Compiles your project and outputs the final executable binary to the `target/` or `build/` directory without running it.

## Testing & Quality Assurance

### 1. `toka test`
A fully automated, zero-configuration test runner. 
Just place your test files inside a `tests/` directory at the root of your project. Running `toka test` will discover all `.tk` files, compile them against your project's `lib/` directory, and output a beautiful, color-coded test report.

```bash
toka test

# Output:
# Running Toka tests...
#   Testing tests/pass/math_test.tk ... [OK]
# Test Results: 1 passed, 0 failed
```

### 2. `toka fmt`
Automatically formats all Toka code in your project. It safely traverses `src/`, `lib/`, `tests/`, and `Project.tk` to ensure your codebase strictly adheres to the official Toka Style Guide.

## Documentation

### `toka doc`
An automated API documentation generator.
Simply document your public functions and shapes using the `///` prefix. Running `toka doc` will extract these signatures and generate a perfectly formatted `docs/API.md` file at the root of your project.

```toka
/// Calculates the sum of two numbers.
pub fn add(a: i32, b: i32) -> i32 {
    return a + b
}
```

## Package Management & Registry (New in v0.9.7)

Toka v0.9.7 introduces the **Official Toka Package Manager**. All package metadata is managed cleanly via the `package.tk` file using the native TON (Toka Object Notation) format.

### `toka search <keyword>`
Instantly searches the official Toka Registry (`pkg.tokalang.dev`) for packages. 
Built on a high-performance, zero-ops edge CDN, it safely parses the registry catalog directly in-memory.

```bash
toka search ink

# Output:
# Search Results:
# ---------------
# toka-ink (0.2.1)
#     Terminal UI component library for Toka language
```

### `toka publish`
Packages your current project and publishes it to the official Toka Package Registry.
*Note: Ensure your `package.tk` has a valid `name`, `version`, and `description` before publishing.*

### `toka add <url_or_name>`
Adds a remote Toka package dependency to your project and updates `package.tk`.

## Housekeeping

### `toka clean`
Instantly removes all temporary compilation artifacts, wiping the `target/`, `build/`, `.toka_build_exe`, and dependency cache directories to free up disk space.
