In the [Shapes](./shapes) chapter, we learned that adding the pipe (`|`) branch separator to a `shape` frees it from being a standard product type, transforming it into a sum type (where multiple variants share memory).

Toka acts strictly depending on whether these are **Tagged** or **Untagged** Variants. They feature vastly different access safety requirements.

## 1. Pure Enum (Tagged)

This is the most common scenario. The moment any branch is merely a variant name, or uses the standard struct format `B(T)` for an anonymous payload, the compiler automatically generates an underlying discriminator (Tag), elevating it to a safe Enum.

```toka
shape Option<'T> (
    Some(val: 'T) |
    None
)

// Use the double colon to construct
auto o = Option<i32>::Some(val = 42)
```

### Destructuring via `match` is Mandatory (Temporary Borrow Guard)

**For memory safety, Toka absolutely forbids accessing Enum members using the dot operator (e.g., `o.val`)!**
Because memory is shared, blindly accessing data if the internal state actually holds a `None` will immediately cause a segmentation fault crash.

To access Data inside an Enum, you **MUST** use a `match` pattern match or an `if auto` guard:

```toka
pub fn check_option(o: Option<i32>) {
    match o {
        // Toka builds a temporary Scope here.
        // `val` is a local variable safely deduced by the compiler.
        auto Some(val) => {
            println("We safely extracted {}", val)
            // As we leave the curly brace, `val` is automatically invalidated.
        }
        None => {
            println("It is an empty value")
        }
    }
}
```

## 2. Untagged Reinterpreted Union

When ALL branches utilize the `as Type` or `as name: Type` **reinterpretation map** semantics, the compiler downgrades the structure into an extremely dangerous but highly efficient underlying C-Union. No extra tags are allocated here.

This is widely used for interoperating with C network libraries, advanced pointer-punning hacking, or extreme memory-compression scenarios.

```toka
// IEEE 754 float reinterpretation hack example
shape FloatInt (
    as f: f32 |
    as i: i32
)

auto u = FloatInt()
u as f32 = 3.14159  // Treat this memory as a float and write to it
auto bits = u as i32 // Instantly extract the identical bit sequence as an i32!
```

### Extreme Danger: Unions CANNOT Hold Pointers
To guard against dangling pointers and illegal release bounds, Toka's semantic analyzer (Sema) **strictly bans** storing pointers (like Exclusive `^`, Borrowed `&`, or structs holding pointers) inside an Untagged `Union`. Such attempts yield a red-card compile rejection!
If you must overlap pointers, use a Tagged `Enum` which provides safe `match` destructors.
