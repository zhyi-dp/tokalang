title: Tuples & Arrays
description: Fixed-size and dynamic collections in Toka.
---

Toka provides built-in support for collections, ranging from basic fixed-size arrays and tuples to more flexible slices and dynamic vectors.

## Tuples

Tuples are fixed-size, heterogeneous collections of values. They are defined using parentheses.

```toka
auto t = (10, "Hello", 3.14)
auto (val, msg, p) = t // Destructuring
```

## Arrays

Arrays are homogeneous collections of values. Fixed-size static arrays can be created directly using square brackets.

```toka
auto arr = [1, 2, 3, 4, 5] // Fixed-size 5 elements
```

## First-Class Dynamic Arrays (Slices)

In Toka, `[T]` represents a First-Class Dynamic Array underlying sequence. Depending on pointer morphology, it represents different memory structures as a **Fat Pointer** (containing both pointer and runtime length):

- `^[T]` (Unique Array): A dynamically allocated array that strictly owns data, and drops sequentially using runtime bonds.
- `&[T]` (Reference Slice): A borrowed view into a sequence. 
- `~[T]` (Shared Array): A reference-counted dynamic array slice block.

```toka
auto ^slice# = new [4]i32(0) // Allocate a 4-element dynamic array

pub fn sum(&data: [i32]) -> i32 { // Pass safely without ownership
    auto res# = 0
    for x in data {
        res += x
    }
    return res
}
```

## Special Syntax: for-or

Toka's `for` loop has a unique `or` clause that executes if the collection is empty.

```toka
for x in data {
    import std/io::println
    println("Value: {}", x)
} or {
    println("Collection is empty!")
}
```

## Standard Library: Vec

While arrays are fixed-size, the standard library provides `Vec<T>` for dynamic resizing.

```toka
import std/vec::Vec

pub fn main() -> i32 {
    auto v# = Vec<i32>::new()
    v.push(10)
    v.push(20)
    return 0
}
```
