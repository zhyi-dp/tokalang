Toka is a modern systems programming language designed for **performance, safety, and conciseness**. It bridges the gap between high-level productivity and low-level control through its unique **Attribute Token System** and **Pointer Morphology**.

## Core Philosophy

Toka is built on a few "Hard Rules" that make code visually predictable and highly distinguishable.

### 1. The Hat Principle (Soul & Identity)
Unlike C or Rust, Toka treats variable names as the **Soul (Value)**. Accessing a pointer variable `p` automatically dereferences it to its data. To operate on the **Identity (Address)**, you must "put a hat on it" using a sigil like `^`, `~`, `&`, or `*`.

- `p = value` (Modify data pointed by p - Soul operation)
- `^p = ^q` (Rebind pointer p to a new identity - Identity operation)

### 2. Conciseness with Attribute Tokens
Toka uses concise tokens to define mutability and nullability. These are discrete dimensions:

- `#` (Suffix to name): **Soul Mutability** (e.g., `auto x# = 10`)
- `?` (Suffix to type): **Entity Nullability** (e.g., `auto x: i32? = none`)
- `nul` (Prefix to name): **Pointer Nullability** (e.g., `auto nul ^p: User = null`)
- `^#`, `~#` (Prefix to name): **Handle Mutability/Rebindability** (e.g., `auto ^#p = new i32(10)`)

### 3. Explicit Ownership
Toka's Pointer Morphology defines ownership and lifetime:
- `^T`: **Unique** ownership (RAII).
- `~T`: **Shared** ownership (Reference Counting).
- `&T`: **Borrowed** access.

## Modern Features

### Pattern Matching
Powerful `match` expressions support Enums, Strings, Integers, and **Or-Patterns** (`|`).

```toka
match result {
    Ok(val) => println("Success"),
    Err("timeout" | "disconnected") => println("Retry required"),
    _ => println("Fatal error")
}
```

### Standard Bitwise Operators
Standard C-style symbols for bitwise logic: `&`, `|`, `^`, `~`, `<<`, `>>`.

## A Quick Taste of Toka

```toka
import std/io::println

// Define a shape (struct/adt)
shape Rect (
    width: i32,
    height: i32
)

// Implement methods
impl Rect {
    pub fn area(self) -> i32 {
        return self.width * self.height
    }
}

pub fn main() -> i32 {
    // Initializing a shape
    auto r = Rect(width = 10, height = 20)
    println("Area: {}", r.area())

    // Arrays and for-or
    auto list = [1, 2, 3]
    for x in list {
        println("Item: {}", x)
    } or {
        println("List is empty!")
    }

    return 0
}
```

<p class="slogan">Toka is built for developers who want the power of C++ with the safety of Rust and the beauty of a modern syntax.</p>
