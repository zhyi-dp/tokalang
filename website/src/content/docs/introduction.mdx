Toka is a modern systems programming language designed for **performance, safety, and conciseness**. It bridges the gap between high-level productivity and low-level control through its unique **Attribute Token System** and **Pointer Morphology**.

## Key Philosophy

Toka is built on several "Iron Laws" that make the code predictable and visually distinct.

### 1. The Hat Principle (Soul vs. Handle)
Unlike C or Rust, Toka treats the variable name as the **Soul** (the value itself). Accessing a pointer variable `p` automatically gives you the data. To operate on the **Handle** (the pointer address), you "put on a hat" using symbols like `^`, `~`, `&`, or `*`.

- `p = value` (Modifies the data pointed to by p)
- `^p = ^q` (Rebinds the pointer p to a new address)

### 2. Syntactic Attributes
Instead of verbose keywords like `mutable`, `optional`, or `volatile`, Toka uses concise tokens:
- `#` for **Mutability** (e.g., `auto x# = 10`)
- `?` for **Nullability** (e.g., `auto x: i32? = none`)
- `#?` for both (e.g., `auto x#? = 10`)

### 3. Explicit Ownership
Toka's Pointer Morphology defines ownership at the type level:
- `^T`: **Unique** ownership (RAII).
- `~T`: **Shared** ownership (Reference Counting).
- `&T`: **Borrowed** access.

## A Quick Taste of Toka

Here is a simple example showing shapes, traits, and the unique `for-or` loop.

```toka
import std/io::println

// Define a shape (struct/adt)
shape Rect (
    width: i32,
    height: i32
)

// Implement methods
impl Rect {
    pub fn area(self) -> i32 {
        return self.width * self.height
    }
}

pub fn main() -> i32 {
    auto r = Rect(width = 10, height = 20)
    println("Area: {}", r.area())
    
    auto list = [1, 2, 3]
    for x in list {
        println("Item: {}", x)
    } or {
        println("List is empty!")
    }
    
    return 0
}
```

Toka is currently under active development, targeting LLVM 20. It's built for developers who want the power of C++ with the safety of Rust and the beauty of a modern syntax.
