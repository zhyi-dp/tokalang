Toka provides native, high-performance support for asynchronous programming using a zero-overhead `async/await` model.

## Core Syntax

Asynchronous functions are defined using the `async` keyword and return a `Task<T>`.

```toka
async fn fetch_data() -> String {
    // simulated async operation
    return "Data fetched!"
}

fn main() {
    auto t = fetch_data()
    auto result = await t
    println(result)
}
```

## The Reactor Model

Toka's async runtime is built on an efficient **Reactor** model (using `epoll` on Linux and `kqueue` on macOS). This allows thousands of concurrent tasks to run on a single thread without the overhead of context switching between OS threads.

### Non-blocking I/O

The standard library provides asynchronous versions of common I/O operations:

```toka
import std::net::TcpStream

async fn handle_connection(stream: ^TcpStream) {
    auto #buf = [0u8; 1024]
    auto n = await stream.read(buf)
    await stream.write(buf.slice(0, n))
}
```

## Memory Safety in Async

Toka's borrow checker is **async-aware**. It ensures that references passed into or held across an `await` point remain valid.

1. **Unique Ownership**: You can move a `^T` into an async closure or task, ensuring it's dropped only when the task completes.
2. **Borrowed Data**: Borrows across `await` points are tracked to prevent use-after-free in asynchronous flows.

## Concurrency Primitives

Toka also provides traditional concurrency tools that work seamlessly with the async model:
- `Mutex<T>`: For protecting shared state.
- `MPSC Channels`: For message passing between tasks.
- `Atomics`: For lock-free operations.

```toka
import sync::mpsc

fn main() {
    auto (tx, rx) = mpsc::channel<i32>()
    
    // Spawn an async task
    spawn async {
        tx.send(42)
    }
    
    auto val = await rx.recv()
    println("Received: {}", val)
}
```
