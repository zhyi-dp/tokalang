title: Async Programming

Toka provides native, high-performance support for asynchronous programming using a zero-overhead `async/await` model.

## Core Syntax

Asynchronous functions are defined using the `fn` keyword, with the `async` modifier applied to the return type.

```toka
fn fetch_data() -> async String {
    // simulated async operation
    return "Data fetched!"
}

pub fn main() -> async i32 {
    auto t = fetch_data()
    auto result = t.await
    println(result)
    return 0
}
```

## The Reactor Model

Toka's async runtime is built on an efficient **Reactor** model (using `epoll` on Linux and `kqueue` on macOS). This allows thousands of concurrent tasks to run on a single thread without the overhead of context switching between OS threads.

### Non-blocking I/O

The standard library provides asynchronous versions of common I/O operations through the `std/task` and `std/net` modules.

```toka
import std/net::TcpStream
import std/task::sleep

fn handle_connection(^stream: TcpStream) -> async void {
    auto buf# = [0u8; 1024]
    auto n = stream.read(buf).await
    stream.write(buf.slice(0, n)).await
}
```

## Starting and Spawning

You can start a task immediately or spawn it into the background.

```toka
fn main() -> i32 {
    fetch_data().start // Fire and forget
    
    // Blocking on the main event loop
    block_on<i32>(fetch_data().to_i32())
    return 0
}
```

## Memory Safety in Async

Toka's borrow checker is **async-aware**. It ensures that references passed into or held across an `await` point remain valid.

1. **Unique Ownership**: You can move a `^T` into an async task using `cede`, ensuring it's dropped only when the task completes.
2. **Borrowed Data**: Borrows across `await` points are tracked to prevent use-after-free in asynchronous flows.
