---
title: 安装指南
description: 如何安装 Toka 编译器及构建工具。
---

为了获取极致的开箱即用体验，Toka 提供了官方预编译的二进制安装包，目前已**全平台原生支持 Windows、macOS 与 Linux**！得益于 Toka 内部静态集成的底层链接器（LLD），您无需配置任何外部 C++ 编译环境即可完成最终可执行文件的链接编译。

## 极速安装 (macOS & Linux)

如果您使用的是 macOS 或 Linux 操作系统，请直接在终端中运行以下命令：

```bash
curl -fsSL https://tokalang.dev/install.sh | bash
```

安装脚本会自动识别您的操作系统与硬件架构，拉取最新版编译器、包管理工具及标准库组合包至 `~/.toka/`，并极其贴心地将 `PATH` 与 `TOKA_LIB` 环境变量自动注入您的配置文件。

## Windows 安装

对于 Windows 用户，Toka 提供了原生的 `.exe` 预编译二进制文件：

1. 前往 [GitHub Releases](https://github.com/tokalang/toka/releases) 页面。
2. 下载最新版本中带有 `Windows` 后缀的压缩包。
3. 将压缩包解压到一个您希望的目录（例如 `C:\tokalang`）。
4. 将解压目录下的 `bin` 文件夹添加至您的系统环境变量 `PATH` 中。
5. 将解压目录下的 `lib` 文件夹路径添加至名为 `TOKA_LIB` 的系统环境变量中。

> 💡 **提示:** Windows 版本基于底层的 Win32 API 进行了深度重构和适配，支持原生开发和调试体验！

> [!WARNING]
> **编译器依赖与环境准备 (重要)**：在 Windows 平台下，核心编译器 `tokac.exe` 依赖 **LLVM/Clang 编译器基础设施** 来完成源代码的翻译与最终链接。
> 如果您的 Windows 系统是纯净裸机，在运行 `toka build` 或 `toka run` 时，后台拉起 `tokac.exe` 会因为找不到 LLVM 动态库而报错崩溃（系统错误码为 `0xC0000135`，显示为 STATUS_DLL_NOT_FOUND 或提示找不到 DLL）。
> **解决方法**：您必须在系统中配置 **MSYS2** 编译环境：
> 1. 下载并安装 [MSYS2](https://www.msys2.org)（默认安装到 `C:\msys64`）。
> 2. 打开 **MSYS2 MinGW64 终端**，复制并运行以下命令，装齐底层编译与链接工具链：
>    ```bash
>    pacman -S --noconfirm mingw-w64-x86_64-clang mingw-w64-x86_64-llvm mingw-w64-x86_64-lld mingw-w64-x86_64-polly make
>    ```
> 3. 将 MSYS2 的二进制目录 `C:\msys64\mingw64\bin` 添加至您的 Windows 系统环境变量 `PATH` 中。

> ⚠️ **功能限制提示:** 当前版本的 Windows **暂不支持原生的底层网络开发**（例如 `std/net`、`stdx/net/http`），任何网络监听或连接操作都会直接返回失败。未来版本将引入基于 IOCP 的完整异步网络支持。如果您的项目重度依赖网络，请暂时使用 WSL2、Linux 或 macOS 环境进行开发。

## 快速体验 (Hello World)

安装完成后，您可以直接复制以下代码一键体验 Toka 的魅力：

```bash
# 1. 快速写入一个经典的 Hello World 文件
echo 'import std/io::{println}

fn main() {
    println("Hello, Toka World!")
}' > hello.tk

# 2. 一键编译并自动执行！
toka run hello.tk
```

## 从源码构建 (进阶)

如果您希望参与 Toka 语言底层的 C++ 核心开发，您可以从头构建整个环境。

### 前置依赖

 - **LLVM 20 & LLD 20**: Toka 现已通过静态链接 LLVM 20 及其内置链接器 (LLD) 来实现零外部依赖的原生编译。因此您必须安装对应的开发头文件包。
   - **Ubuntu/Debian**: 需安装 `llvm-20-dev` 和 `liblld-20-dev`。
   - **macOS**: 通过 Homebrew 安装：`brew install llvm@20`（自带 LLVM 及 LLD 组件）。
   - **Windows**: 通过 MSYS2 (MINGW64 环境) 安装：`pacman -S mingw-w64-x86_64-llvm mingw-w64-x86_64-lld`。
 - **CMake**: 3.15 或更高版本（对于 Windows MSYS2 用户，请一并安装 `mingw-w64-x86_64-cmake` 和 `mingw-w64-x86_64-ninja`）。
 - 操作系统对应的 C++ 编译链工具（如 MSVC / GCC / Clang。Windows MSYS2 环境请使用 `mingw-w64-x86_64-clang`）。

### 开始构建

Toka 项目内包含了一个极其方便的自动化构建脚本 `rebuild.sh`，它可以一次性为您编译好 `tokac` (核心编译器) 以及纯正原生的 `toka` (交互式包管理器命令行)。

1. 克隆代码仓库：
   ```bash
   git clone https://github.com/tokalang/toka.git
   cd tokalang
   ```

2. 首次运行前配置 CMake 项目（只需一次）：
   ```bash
   mkdir build && cd build
   cmake ..
   cd ..
   ```

3. 运行根目录下的全量编译脚本：
   ```bash
   ./rebuild.sh
   ```

编译完成后，您的编译器与包管理工具会自动输出并存放在本项目的 `build/bin` 目录下。

## 环境配置

为了在任何地方都可以自由无感地使用 Toka，请将以下几行配置加入到您的 `.zshrc` 或者 `.bashrc` 中：

```bash
# 将刚刚编译出的 tokac 和 toka 所在的产物目录加入环境变量
export PATH="/path/to/tokalang/build/bin:$PATH"

# 关键设置：让编译器及包管理工具精准地定位到标准库
export TOKA_LIB="/path/to/tokalang/lib"

# 第三方库生态设置 (新增)：定义全局依赖搜索路径
# 多个路径可以使用冒号 ":" 分割。你可以将个人的常用库或下载的源码包放进这些目录，
# 然后在代码中就可以直接 `import mylib::{...}` 无缝导入！
export TOKA_PATH="/path/to/your/custom/libs:/another/lib/path"

# (可选) 指定用于底层链接的工具。默认为 macOS 原生或 /usr/local/opt/llvm@20/bin/clang
export TOKA_CLANG="/usr/local/opt/llvm@20/bin/clang"
```

> [!IMPORTANT]
> 请务必将 `/path/to/tokalang` 替换为您本地克隆该工程的实际完整绝对路径。

## 使用包管理工具开发

配置好环境变量后，您可以在任意目录直接使用原生 `toka` 命令来开发应用！

### 1. 初始化新工程

通过 `new` 命令可以快速生成带有标准化目录结构的工程包：

```bash
toka new my_app
cd my_app
```

该命令将为您生成：
- `src/main.tk`: 项目入口文件。
- `toka.mod`: Toka 的声明式包管理与依赖描述清单。您可以通过此文件配置依赖项目和包属性。

### 2. 引入文件与第三方依赖

Toka 提供了一套极其直观、符合直觉的 `import` 路径模块系统：

**导入本地文件（相对与绝对路径）**：
Toka 原生支持直接将路径写进代码里（注意：无需包裹在引号中）：
```toka
// 相对路径导入当前目录的 utils.tk
import ./utils::{foo, bar}
import ../lib/network::{get}

// 绝对路径导入
import /opt/shared/mylib::{baz}
```

**导入第三方包管理依赖**：
如果你通过包管理器 `toka add pkg.tokalang.dev/user/ansi` 安装了第三方包，那么它会被注册。随后你可以直接在代码中使用标准方式导入它：
```toka
import ansi::{...}
```

> **提示:** 编译器在解析裸名称（如 `ansi`）时，会依次从标准库 (`TOKA_LIB`) 以及我们刚刚配置的全局搜索路径 (`TOKA_PATH`) 中进行查找匹配。

### 3. 一键编译与运行

在平时开发测试时，您不需要手动操作编译器，直接在由 `toka.mod` 统控的目录下执行：

```bash
toka run
```

该命令将自动读取配置、下载拉取所有必须的依赖库，完成编译，并最终向您呈现可执行文件的运行输出！如果您只想编译出最终二进制文件而不运行，可以使用 `toka build`。
