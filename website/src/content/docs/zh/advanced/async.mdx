title: 异步编程
description: 通 Toka 的 async/await 实现高性能并发。
---

Toka 通过零开销的 `async/await` 模型提供原生的、高性能的异步编程支持。

## 核心语法

异步函数使用 `fn` 关键字定义，并将 `async` 修饰符应用于返回类型。

```toka
fn fetch_data() -> async String {
    // 模拟异步操作
    return "数据已获取！"
}

pub fn main() -> i32 {
    auto t = fetch_data()
    auto result = t.await
    println(result)
    return 0
}
```

## Reactor 模型

Toka 的异步运行时建立在高效的 **Reactor** 模型之上（在 Linux 上使用 `epoll`，在 macOS 上使用 `kqueue`）。这允许数千个并发任务在单个线程上运行，而无需在 OS 线程之间进行上下文切换。

### 非阻塞 I/O

标准库通过 `std/task` 和 `std/net` 模块提供常用 I/O 操作的异步版本。

```toka
import std/net::TcpStream
import std/task::sleep

fn handle_connection(^stream: TcpStream) -> async void {
    auto buf# = [0u8; 1024]
    auto n = stream.read(buf).await
    stream.write(buf.slice(0, n)).await
}
```

## 启动与派生

你可以立即启动一个任务，或者将其派生到后台运行。

```toka
fn main() -> i32 {
    fetch_data().start // 即刻启动（Fire and forget）
    
    // 在主事件循环上阻塞
    block_on<i32>(fetch_data())
    return 0
}
```

## 异步中的内存安全

Toka 的借用检查器是 **异步感知的 (Async-aware)**。它确保在 `await` 点传递或持有的引用保持有效。

1. **唯一所有权**: 你可以使用 `cede` 将 `^T` 移动到异步任务中，确保它仅在任务完成时才被注销。
2. **借用数据**: 跨 `await` 点的借用会被追踪，以防止异步流中的使用后释放（use-after-free）问题。
