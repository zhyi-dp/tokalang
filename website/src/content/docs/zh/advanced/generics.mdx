---
title: 泛型 (Generics)
description: 掌握如何使用 Toka 的尖括号构建可复用的编译期多态组件。
---

泛型允许你编写可处理多种类型的函数和数据结构，而不会带来运行时的性能损耗。Toka 编译器使用**单态化（Monomorphization）**策略，在编译期为你调用的每一种数据类型生成一份专用的高性能机器码。

## 泛型数据结构
在 `shape` 名称后面追加尖括号即可定义泛型组件。

```toka
// 一个非常实用的包装器
shape Result<'T, 'E> (
    Ok(: 'T) |
    Err(: 'E)
)

// 使用
auto res = Result<i32, String>::Ok(42)
```
> [!TIP]
> 泛型参数名称（例如 `'T`、`'E`）通常建议以单引号起头，这表示它们是一个尚未确定的类型占位符。

## 泛型函数
函数也可以使用尖括号定义泛型签名：

```toka
pub fn identity<'T>(val: 'T) -> 'T {
    return val
}

auto x = identity<i32>(100)
```
在很多情况下，Toka 的类型推演引擎能根据入参的类型自动推断出 `<'T>` 是什么，你只需要写 `identity(100)` 即可。


## 泛型上的方法扩展 (impl)
你可以只针对特定的泛型实例写入扩展。比如如果我要给 `Mutex` 补充方法：

```toka
// 这表示你将为所有 T 创建方法
impl<'T> Mutex<'T> {
    pub fn do_something() {}
}

// 这表示你只为装有 i32 类型的 Mutex 增加特权方法，
// 其他 T 类型的 Mutex 无法调用此方法。
impl Mutex<i32> {
    pub fn do_i32_specific_math() {}
}
```

## Trait 约束 (Bound Constraints)
泛型一旦脱离约束，就只能执行基础的参数转移操作。如果你想在泛型函数里调用某个具有前提的方法，诸如 `val1 == val2` 或者 `val.hash()`，你必须在尖括号里显式要求实参必须符合这类标准。

这正是 [接口与特征 (Traits)](../types/traits) 发挥威力的舞台。

```toka
// 只有实现并满足了 @PartialEq 特征的两个类型，
// 这个函数里才能安全地执行 val1 == val2，这避免了非常多的未定义行为。
pub fn is_equal<'T: @PartialEq>(val1: 'T, val2: 'T) -> bool {
    return val1 == val2
}
```

## 泛型模块内的最高权限屏障 `@encap`

在封装一个极其危险或严谨的系统时，你可以给任意的泛型系统加盖 `@encap` (封装/隐藏) 印章。这指示编译器将 `shape` 内所有变量（如 `data`, `root_ptr` 等）的访问级别立刻锁定为**私有**。

```toka
impl<'T> Vec<'T>@encap {
    pub(std) * // 给同样身处标准库 std 的兄弟开辟访问绿灯，外部开发者谢绝入内
    
    // 如果没有留 pub fn 给你提供内部指针，任何开发者在他们的代码中尝试
    // p.buf 都会在编译阶段被 Toka 收缴。
}
``` 
这对于涉及到底层指针释放和堆操作的并发安全容器来说是极为重要的安全堡垒。
