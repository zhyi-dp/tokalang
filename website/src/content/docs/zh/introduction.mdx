Toka 是一门为 **性能、安全和简洁** 而设计的现代系统级编程语言。它通过独特的 **属性令牌系统 (Attribute Token System)** 和 **指针构词学 (Pointer Morphology)**，在高效的底层控制与高层生产力之间架起了一座桥梁。

## 核心哲学

Toka 建立在几项“铁律”之上，使代码在视觉上具有高度的可预测性和区分度。

### 1. 戴帽原则 (灵与肉)
与 C 或 Rust 不同，Toka 将变量名视为 **Soul (灵/值本身)**。直接访问指针变量 `p` 会自动解引用得到数据。若要操作 **Handle (肉/指针地址本身)**，你需要通过符号（如 `^`, `~`, `&` 或 `*`）为其“戴上帽子”。

- `p = value` (修改 p 指向的数据内容)
- `^p = ^q` (将指针 p 重绑定到一个新的地址)

### 2. 语法令牌属性
Toka 不使用冗长的关键字（如 `mutable` 或 `optional`），而是使用简洁的令牌作为后缀：
- `#` 表示 **可变性** (例如：`auto x# = 10`)
- `?` 表示 **可空性** (例如：`auto x: i32? = none`)
- `#?` 表示 **既可变又可空** (例如：`auto x#? = 10`)

### 3. 显式所有权
Toka 的指针构词学在类型层面定义了所有权关系：
- `^T`：**独占**所有权 (基于 RAII)。
- `~T`：**共享**所有权 (基于引用计数)。
- `&T`：**借用**访问。

## Toka 代码初探

下面是一个展示 Shape 定义、方法实现以及独特的 `for-or` 循环的简单示例。

```toka
import std/io::println

// 定义一个 Shape (结构体) - 必须使用圆括号
shape Rect (
    width: i32,
    height: i32
)

// 实现固有方法
impl Rect {
    pub fn area(self) -> i32 {
        return self.width * self.height
    }
}

pub fn main() -> i32 {
    auto r = Rect(width = 10, height = 20)
    println("面积: {}", r.area())
    
    auto list = [1, 2, 3]
    for x in list {
        println("项: {}", x)
    } or {
        println("列表为空！")
    }
    
    return 0
}
```

Toka 目前正在积极开发中，目标支持 LLVM 20。它专为那些既渴望 C++ 的强大动力，又追求 Rust 的安全性与现代语法美感的开发者而设计。
