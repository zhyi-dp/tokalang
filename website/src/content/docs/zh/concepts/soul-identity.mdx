---
title: 灵魂与肉体 (Soul vs Handle)
description: Toka 语言类型系统的哲学核心。
---

在 Toka 中，每个对象都有两个截然不同的层面：它的 **Soul (灵魂/值)** 和它的 **Handle (肉体/指针/标识)**。理解这两者的平衡是掌握 Toka 内存模型的关键。

## 灵魂 (Soul / 值)
“Soul” 代表对象的实际数据或状态。当你对 soul 执行操作时，你是在修改其内容。

- **访问方式**: 直接使用变量名 `p`。
- **属性令牌**: `#` (可写) 和 `?` (可空) 令牌作为后缀应用于变量名或类型时，会影响 Soul。

```toka
auto x# = 10
x = 20 // 修改了 x 的灵魂（数据本身）
```

## 肉体 (Handle / 指针 / 标识)
“Handle” 代表对象在内存中的位置。当你对 handle 执行操作时，你是在改变其标识（即指针指向哪里）。

- **访问方式**: 通过变量名加“帽子”（Sigil），如 `^p`, `~p`, `&p`, 或 `*p`。
- **构词 Sigils**: 这些符号定义了 Handle 的属性（独占、共享或借用）。

```toka
auto ^p# = new i32(10)
^p = ^q // 重写了 Handle (p 以后将指向 q 的位置)
```

## “戴帽”原则

许多编程语言模糊了“引用赋值”与“值赋值”之间的界限。

在 Toka 中，这种区别在视觉上是极其鲜明的：
1. `p = q`: **灵魂操作 (Soul Operation)**。将 `q` 的数据拷贝或移动到 `p`。
2. `^p = ^q`: **肉体操作 (Handle Operation)**。重置指针 `p`，使其指向 `q` 所指向的地址。

通过这种显式的区分，Toka 消除了像 C++ 或 Rust 中常见的“这是深拷贝还是浅拷贝？”的歧义。

## 概念对比表

| 概念 | 符号示例 | 核心关注点 | 修饰词 |
| :--- | :--- | :--- | :--- |
| **Soul (灵魂)** | `p` | 数据 / 内容 | `p#` (可变), `p?` (可空) |
| **Handle (肉体)** | `^p`, `~p`, `&p` | 地址 / 所有者 | `nul ^p` (可空), `^p#` (可重绑定) |
