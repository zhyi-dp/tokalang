Toka 引入了 **属性令牌系统 (Attribute Token System)**，这是定义类型和字段行为的一种极其简洁的方式，无需冗长的关键字或复杂的修饰符。

## 三大基础令牌

在 Toka 中，每个类型都可以通过一个或多个属性令牌进行修饰，从而定义其存储和访问属性。这些令牌可以作为基础类型的**后缀**，也可以作为指针的**修饰符**。

### 1. 可写性 (`#`)
`#` 令牌表示一个值或字段是 **可变的 (Mutable)**。在 Toka 中，所有字段和变量默认都是不可变的。你必须显式地用 `#` 标记它们以允许修改。

```toka
shape User (
    #name: String,  // 该字段可以被修改
    id: i32         // 该字段在初始化后是不可变的
)

fn main() {
    auto #x = 10    // 可变变量
    x = 20          // 正常运行
    
    auto y = 10     // 不可变变量
    // y = 20       // 错误：试图为不可变变量赋值
}
```

### 2. 可空性 (`?`)
`?` 令牌表示一个值可以是 **None** (等同于空指针或 Optional 类型)。这强制要求开发者显式处理缺失值的情况，从而避免空指针异常。

```toka
fn find_user(id: i32) -> ?User {
    // ...
    return none
}

fn main() {
    auto ?maybe_user = find_user(1)
    if maybe_user is User {
        // 在这里，maybe_user 会被安全解包
        println("找到用户: {}", maybe_user.name)
    } or {
        // 当返回值为 none 时执行
        println("未找到用户")
    }
}
```

### 3. 可写且可空 (`!`)
`!` 令牌是 `#` 和 `?` 的合并缩写（即 `#?`）。它表示一个字段既可以被重新赋值，也可以被设为 `none`。这对于那些在初始化时刻可能为空并在后续赋值的字段非常有用。

```toka
shape Session (
    !current_user: User // 既可以被修改，也可以设为 none
)
```

## 属性的“叠加态”

属性令牌可以叠加使用，并应用于不同的嵌套层级，如同化学分子的排列组合：

- `i32#`: 一个可变的整数。
- `^User?`: 一个可为空的独占指针，指向一个不可变的 User。
- `^#User!`: 一个可变的独占指针，指向一个既可变又可空的 User。

这种“化学分子式”式的语法确保了即使是嵌套极深的复杂类型也能在一瞥之间被理解。

## 编译器实现细节

在编译器的内部实现中，这些属性以位掩码的形式存储在类型系统（`Type.cpp`）中。语义分析阶段会进行严格检查，以确保：
- 没有任何向不可变位置写入的操作。
- 在访问可空类型前必须进行空值检查（`is` 或 `if-not`）。
- 属性会在泛型替换过程中被正确继承和传播。
