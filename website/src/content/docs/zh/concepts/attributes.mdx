---
title: 语法属性令牌 (Attribute Tokens)
description: Toka 语言中的可变性与可空性令牌。
---

Toka 语言不再使用冗长的关键字（如 `mutable` 或 `optional`），而是采用一套简洁的属性令牌系统。这些令牌是离散的维度，可以自由组合，用于定义 Soul（灵魂）和 Handle（肉体）的精确行为。

## 离散的维度 (Discrete Dimensions)

### 1. Soul 可变性 (`#` 后缀)
当 `#` 作为变量名或字段名的后缀时，表示其 **Soul (数据本身)** 是可变的。

```toka
auto x# = 10
x = 20 // 允许执行：修改了 x 的灵魂（数据本身）
```

### 2. 实体可空性 (`?` 后缀)
当 `?` 作为 **类型 (Type)** 的后缀时，表示该实体的值可以为 `none`。

```toka
auto x: i32? = none
if x == none {
    println("无值")
}
```

### 3. 指针本身可空 (`nul` 前缀)
`nul` 关键字作为指针名称的前缀时，表示该 **Handle (指针本身)** 可以为 `null`。

```toka
auto nul ^p: User = null
if ^p == null {
    println("无指针")
}
```

### 4. Handle 可重绑定性 (`^#`, `~#` 等)
在指针符 (Sigil) 之后跟随 `#` 令牌，表示该 **Handle (指针本身)** 是可写的（即可以重绑定到不同的地址）。

```toka
auto ^#p = new i32(10)
^p = ^q // 允许执行：重绑定了 Handle
```

## “分子式”组合方式

你可以像书写化学分子式一样，通过叠加这些令牌来定义复杂的行为。

### 示例：一个可空且可重绑定的指针，指向一个可变且可空的 Node 实体

```toka
auto nul ^#node#: Node? = null
```

拆解说明：
1. `nul`: 指针本身可以为 `null`。
2. `^#`: 这是一个独占指针，且指针本身支持重写 (`^node = ^other`)。
3. `node#`: 变量名后的 `#` 表示指向的数据（灵魂）是可变的 (`node.value = 10`)。
4. `Node?`: 指向的实体类型允许为 `none`。

## 属性对比表

| 令牌 | 位置 | 作用维度 | 效果 |
| :--- | :--- | :--- | :--- |
| **`#`** | 名称后缀 | Soul 可变性 | 允许修改数据内容 |
| **`?`** | 类型后缀 | 实体可空性 | 允许值为 `none` |
| **`nul`** | 名称前缀 | 指针可空性 | 允许指针为 `null` |
| **`^#`** | 名称前缀 | Handle 可变性 | 允许指针重绑定 (Rebind) |
