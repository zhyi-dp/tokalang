Toka 的内存安全模型深深刻在它的 **指针构词学 (Pointer Morphology)** 中。
与使用 `Box<T>` 或 `Rc<T>` 这种泛型容器不同，Toka 使用特定的 **Sigils (符号)** 直接定义指针与其指向数据之间的所有权和生命周期关系。

## 指针类型详解

Toka 提供了四种主要的指针类型，每种类型都有独特的所有权和生命周期语义。

### 1. 独占指针 (`^`)
`^` 符号表示 **排他性所有权**。在任一时刻，一个数据只能由一个独占指针拥有。
- **RAII**: 当 `^` 离开作用域时，它所拥有的内存会自动释放。
- **移动语义 (Move)**: 默认进行所有权转移。使用 `cede` 关键字进行显式移交。

```toka
pub fn delete_user(^u: User) {
    // u 将在该函数结束时被自动注销 (Drop)
}

pub fn main() -> i32 {
    auto ^u = new User(name = "Alice")
    delete_user(cede u) // 使用 cede 显式移交所有权给函数
    // 此时 u 在此处已不可再用
    return 0
}
```

### 2. 共享指针 (`~`)
`~` 符号表示 **基于引用计数的共享所有权**。
- **自动化**: 引用计数由编译器自动维护，在拷贝时递增，在销毁时递减。
- **安全性**: 只要还有至少一个 `~` 存在，数据就会保持存活。

```toka
shape Node (
    data: i32,
    nul ~next: Node // 共享且可为空的指针
)
```

### 3. 借用 / 引用 (`&`)
`&` 符号用于 **借入 (Borrowing)**。它提供对数据的临时、非所有权视图。
- **安全性**: 借用检查器 (Borrow Checker) 确保借用的生命周期不会超过其所有者的生命周期。
- **可写性**: 使用 `&p#` 表示一个指向可变实体的借用。

```toka
pub fn print_name(&u: User) {
    println("用户名: {}", u.name)
}
```

### 4. 原始指针 (`*`)
`*` 符号代表 **原生的 C 风格指针**。
- **不安全**: 仅在 FFI（调用 C 库）或极底层内存操作时使用。

## 可重绑定与实体的可变性

Toka 严格区分了 **指针本身的重定向 (Rebindability)** 与 **指向实体的读写权限 (Writability)**。

| 代码示例 | 语义描述 |
| :--- | :--- |
| `auto ^p` | 只读的独占指针。 |
| `auto ^p#` | 指向可变实体的独占指针。 |
| `auto &#p` | 可重绑定的借用指针（指针地址本身可变）。 |
| `auto nul ^#node#: Node?` | 一个指向可空、可变实体，且其指针本身也可重绑定的可空独占指针。 |

## 戴帽原则 (灵魂与肉体)

在 Toka 中，裸名变量 `p` 代表 **Soul (灵魂/值本身)**。直接访问 `p` 会自动解引。
若要操作 **Handle (肉体/指针地址本身)**，你需要通过符号“戴上帽子”。

- `p = value` (修改灵魂：操作 p 指向的数据)
- `^p = ^q` (修改肉体：操作 p 指针本身的指向)

这种设计使得值操作与地址操作在视觉上极其鲜明，降低了理解底层逻辑的负担。
