Toka 的 **指针构词学 (Pointer Morphology)** 通过附加在变量名或字段名上的简洁符号（Sigils），定义了对象的标识（Identity）和所有权性质。

## 指针类型 (Sigils)

Toka 使用三种主要的指针符号：

### 1. 独占指针 (`^`)
表示对数据的唯一所有权。当一个 `^T` 离开作用域时，内存会被自动释放 (RAII)。

```toka
auto ^p = new i32(10)
```

### 2. 共享指针 (`~`)
表示通过线程安全的引用计数实现的共享所有权。

```toka
auto ~p = new i32(20)
auto ~q = ~p // 引用计数增加
```

### 3. 借用指针 (`&`)
表示对由他人拥有的数据的非所有权访问。

```toka
auto x = 10
auto &p = &x
```

## Handle 属性

指针本身（Handle）可以通过前缀获得额外的控制。

### 指针本身可空 (`nul`)
`nul` 前缀允许一个指针的值为 `nullptr`。

```toka
auto nul ^p: User = nullptr
```

### Handle 可重绑定性 (`#`)
在符号后添加 `#` 允许指针变量本身被重新赋值（Handle Rebinding）。

```toka
auto ^#p = new i32(1)
^p = ^q // 允许执行：Handle 是可重绑定的
```

## “分子式”叠加

你可以在任何指针上叠加这些维度。

```toka
auto nul ^#node#: Node? = nullptr
```

- `nul`: 指针本身可以为 `nullptr`。
- `^#`: 这是一个独占指针，且指针变量本身可以被重写 (`^node = ^other`)。
- `node#`: 指向的数据（Soul）是可变的。
- `Node?`: 实体本身可以为 `none`。

## 自动解引用

Toka 遵循 **“戴帽原则”**：如果你直接访问变量名 `p`，你就是在访问 **Soul (数据)**。编译器会自动为你解引用。

```toka
auto ^p = new User(name = "Alice")
println(p.name) // 自动解引用以访问 name
p.name = "Bob" // 自动解引用以修改 name (如果 User 的 Soul 是可变的)
```
