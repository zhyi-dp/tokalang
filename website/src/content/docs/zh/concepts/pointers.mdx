title: 指针与帽子法则 (Pointers & Hat Principle)
description: Toka 独创的四种核心指针形态与深刻的灵壳分离哲学。
---

Toka 并没有采用传统的 C/C++ 或 Rust 那样的星号(`*`)与与号(`&`)组合语法，而是彻底重构了内存标识系统，提出了著名的 **“帽子法则 (Hat Principle)”**。

在 Toka 的设计哲学中，**“形态即身份，脱帽即灵魂”**。

## 一、 核心指针形态 (The Four Hats)

Toka 不允许隐式指针操作或隐藏引用。任何指针变量在代码中出场时，必须在任何地方携带其形态符号（“帽子”），否则会自动触发解引用（脱帽）。

| 符号 | 形态名称 | 语义逻辑与场景 |
| --- | --- | --- |
| **`^`** | **独占指针 (Unique)** | 具有绝对所有权，自动析构 (RAII)，赋值时隐式产生 `move` 并夺取所有权。常用语托管对象的生命周期根节点。 |
| **`~`** | **共享指针 (Shared)** | 基于内置底层引用计数的共享所有权，赋值时增加计数。常用于跨线程或图结构的节点管理。 |
| **`*`** | **裸指针 (Raw)**    | 手动接管内存地址，不具备任何自动管理与所有权语义。必须在 `unsafe` 环境下操作。 |
| **`&`** | **引用 (Reference)**| 具有明确时效的借用观察者，**不代表取地址**！安全冻结目标所有者的权限直至观察期结束。 |

> [!WARNING]
> 与 C 家族不同，Toka **绝对禁止** 在变量名前出现帽子叠加，如 `**p` 或 `^&p` 均是非法的语法。

### 形态依附于变量名，而非类型

一个非常重要且强烈的 Toka 特色是：**指针符号始终佩戴在变量名头上，而非类型上。** （这也解释了为什么叫它为帽子）。

```toka
// 正确 (Toka 风格)
auto ^p: Node = new Node() 

// 错误 (传统 C/Rust 思维，将被 Toka 编译器拒绝)
auto p: ^Node = new Node()
```

## 二、 灵与壳的剥离 (Soul vs Handle)

为何指针符号要加在变量前面？这源自 Toka 的核心准则：

- **戴帽子 `(*p, ^p, ~p, &p)`**：你触碰的是 **载体本身 (Handle)**。也就是指针自己。改变它是**重绑定指针的指向**。
- **摘帽子 `(p)`**：你触碰的是 **灵魂本体 (Soul)**。即真实的数据结构。操作它是为了**直接读写目标内存**。

### 颠覆性的反向思维

习惯 C/C++ 的开发者请注意，Toka 的解引用逻辑恰好与 C 语言翻转：

**在 C++ 中：**
- `p` 是指针本体
- `*p` 是解引用后的数据

**在 Toka 中：**
- `*p` (或 `^p` / `~p`) 是指针本体（Handle）
- `p` 是解引用后的数据（Soul）

### 代码示例：身份守恒的同频赋值

因为形态代表了身份，Toka 要求等号左右两侧的帽子状态必须**视觉对称**：

```toka
auto ^p = new i32(42)

// 1. 操作 Handle（重绑定指针自身指向）
auto ^q = new i32(100)
^p = cede ^q     // 戴着帽子交换控制权！^p 获得 ^q 的所有权，原 ^p 的内容自动 Drop。

// 2. 操作 Soul（读写最终的数据）
p = 200          // 摘下帽子，直接将 200 写入 p 指向的内存，也就是解引用写入。
```

> [!NOTE]
> 这意味着，只要你在代码中看到没有戴帽子的裸名 `p`，你就可以绝对放心地将其像普通值类型一样进行操作和字段访问（如 `p.x`），编译器早已经在这个字符背后帮你完成了安全的自动解引用与溯源。

## 三、 内存的降维与借用

### 1. 安全借用投射 (`as`)
如果要安全地观察一个 `^` 或 `~` 拥有的数据，我们可以使用安全的借用投影。
假设存在 `^q`：

```toka
// as 充当安全转换投影的算子
auto &p = ^q as &Node
```
这一瞬间，`^q` 进入冻结 (FROZEN) 状态。直到观察者 `&p` 的生命周期结束之前，任何人无法修改原本的结构体。

### 2. OAddr：万能的外挂观察片
如果你仅仅是需要拿到一个数值化的内存地址来做 Log 打印或者 C-FFI 对接，但完全不涉及生命周期：

```toka
// 获取裸指针本身的数值化地址 usize，可供打印
println("Pointer address: 0x{:X}", *raw as OAddr) 
```
