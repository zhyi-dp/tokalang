Toka 的内存安全模型深深刻在它的 **指针构词学 (Pointer Morphology)** 中。
不同于其他语言使用如 `Box<T>` 或 `Rc<T>` 这样的泛型容器，Toka 采用特定的 **Sigils (符号)** 直接在语法层面定义指针与其指向数据之间的所有权关系。

## 指针类型详解

Toka 提供了四种主要的指针类型，每种类型都有其独特的所有权和生命周期语义。

### 1. 独占指针 (`^T`)
`^` 符号表示 **排他性所有权**。在任何给定的时刻，一个数据只能有一个独占指针指向它。
- **RAII**: 当 `^T` 离开作用域时，它指向的内存会被自动释放。
- **移动语义 (Move)**: 将一个 `^T` 赋值给另一个变量会转移所有权。原始变量将变得不可用。

```toka
fn delete_user(u: ^User) {
    // u 将在该函数结束时被自动销毁 (Drop)
}

fn main() {
    auto ^u = new User(name = "Alice")
    delete_user(u) // 所有权转移给了 delete_user 函数
    // 此时 u 在此处已不可再用
}
```

### 2. 共享指针 (`~T`)
`~` 符号表示 **基于引用计数的共享所有权**。
- **自动化**: 引用计数在拷贝时递增，在销毁时递减。
- **安全性**: 只要还有至少一个 `~T` 存在，指向的数据就会保持存活。

```toka
shape Node (
    data: i32,
    next: ?~Node // 共享且可为空的指针
)
```

### 3. 借用 / 引用 (`&T`)
`&` 符号用于 **借入 (Borrowing)**。它提供对数据的临时、非所有权视图。
- **安全性**: 借用检查器 (Borrow Checker) 确保借用的生命周期不会超过其所有者的生命周期。
- **权限**: 默认情况下，借用是不可变的。使用 `&#T` 表示可变借用。

```toka
fn print_name(u: &User) {
    println("用户名: {}", u.name)
}
```

### 4. 原始指针 (`*T`)
`*` 符号代表 **原生的 C 风格指针**。
- **不安全**: 原始指针不提供任何内存安全保证。
- **FFI**: 主要用于调用 C 库（FFI）或进行底层内存操作。

## 指针与属性的组合

指针符号与属性令牌（Attribute Token）可以自由组合，从而创造出极其精确的类型定义：

| 类型 | 语义描述 |
| :--- | :--- |
| `^User` | 拥有的、不可变的 User。 |
| `^#User` | 拥有的、可变的 User。 |
| `~?User` | 共享的、可为空的、不可变的 User。 |
| `&#User` | 对 User 的可变借用。 |

## “构词学” 的优势

通过使用符号（`^`, `~`, `&`）而非泛型类，Toka 将内存的“物理布局”直接显化在代码中。
这大大降低了阅读复杂系统代码时的认知负担，因为所有权规则在视觉上与普通数据类型完全区分开来。
