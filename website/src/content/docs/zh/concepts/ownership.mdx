---
title: 所有权与 RAII
description: Toka 语言中的资源管理与内存安全。
---

Toka 的内存安全性建立在清晰的 **所有权 (Ownership)** 模型之上，由编译器通过指针构词学 (Pointer Morphology) 和借用检查器 (Borrow Checker) 强制执行。

## 独占所有权 (`^`)

独占指针 (`^T`) 是资源的唯一持有者。当它离开作用域时，资源将被自动清理 (RAII)。

```toka
fn take_ownership(^u: User) {
    println("拥有用户: {}", u.name)
} // u 在此处被释放 (dropped)

pub fn main() -> i32 {
    auto ^user = new User(name = "Alice")
    take_ownership(user) // 所有权转移
    // user 在此之后不再可访问
    return 0
}
```

## 共享所有权 (`~`)

共享指针 (`~p`) 使用引用计数。资源仅在最后一个引用被释放后才会被销毁。

```toka
shape Node (
    value: i32,
    ~next: Node?
)
```

## 借用 (`&`)

借用 (`&p`) 允许你在不取得所有权的情况下访问数据。借用可以是不可变的 (`&p`) 或可变的 (`&p#`)。

```toka
fn print_user(&u: User) {
    println(u.name)
}

pub fn main() -> i32 {
    auto ^user = new User(name = "Bob")
    print_user(user) // 不可变借用 (Immutable borrow)
    return 0
}
```

## 显式转移 (`cede`)

你可以使用 `cede` 关键字显式地转移一个 Soul (灵魂) 的所有权。

```toka
fn process_data(^v: Vec<i32>) { ... }

pub fn main() -> i32 {
    auto ^v = [1, 2, 3]
    process_data(cede v) // v 已被 cede (让渡)
    return 0
}
```

## @encap Trait

对于管理外部资源（如文件句柄或套接字）的类型，实现 `@encap` trait 可以自定义清理逻辑。

```toka
shape File (
    handle: i32
)

impl File@encap {
    pub fn drop(self) {
        // 自定义的关闭逻辑
    }
}
```
