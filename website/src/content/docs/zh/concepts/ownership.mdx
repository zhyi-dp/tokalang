Toka 通过严格的所有权规则和 **RAII (Resource Acquisition Is Initialization)** 模式结合，确保了系统级别的资源安全。

## 所有权规则

所有权在 Toka 中由 **独占指针 (`^`)** 定义。

1. **唯一所有者**: 在任何给定时间，每个资源（内存、文件句柄、套接字）都只有一个唯一的独占指针所有者。
2. **默认移动 (Move)**: 赋值或传递一个独占指针会转移其所有权，使原始指针失效。
3. **显式移交**: 使用 `cede` 关键字可以显式地将所有权移交给闭包或多线程任务。

```toka
fn take_ownership(^u: User) {
    // u 将在此函数结束时被注销 (Drop)
}

pub fn main() -> i32 {
    auto ^u = new User(name = "Bob")
    take_ownership(cede u) // 显式移动所有权
    // 此处 u 已不可再用
    return 0
}
```

## RAII 与 @encap Trait

Toka 使用 `impl Type@encap` 块来定义资源如何被回收。这相当于析构函数或 `drop` 函数。

```toka
shape FileHandle (
    fd: i32
)

impl FileHandle@encap {
    // 编译器会自动合成一个析构机制
    // 当 FileHandle 离开作用域时，会调用此逻辑。
    pub fn drop(self) {
        if self.fd > 0 {
            libc_close(self.fd)
            println("文件句柄已关闭。")
        }
    }
}
```

## 借用 (References)

若要在不获取所有权的情况下使用数据，请使用 **借用 (`&`)**。

1. **安全性**: 借用在任何时候都不能超过其所有者的生命周期。
2. **默认不可变**: `&T` 借用是只读的。
3. **可变借用**: 使用 `&T#` 允许修改其指向的实体数据。

```toka
pub fn process_data(&u: User) {
    // 只读访问
}

pub fn update_user(&u#: User) {
    // 对 User 灵魂（数据）进行修改
    u.name = "新名字"
}
```

## 移动与返回值

当函数返回一个拥有的对象时，它会自动移动给调用者。不会发生昂贵的拷贝或显式的析构。

```toka
fn create_user() -> ^User {
    return new User(name = "Charlie") // 所有权移交给调用者
}
```

这种模型提供了垃圾回收语言的安全性，同时保留了手动内存管理的可预测性能。
