# Toka 1.0 字符串体系设计与正交转换完全体法典

本法典为 Toka 1.0 标准库稳定版中关于字面量、`str`、`String`、`cstr` 以及底层裸指针之间的**终极物理实相、主权边界、与正交转换规范**的最高法律定义。

---

## 🏛️ 一、 核心立宪宗旨

Toka 恪守重工业级系统编程语言的至高设计品味：

> **宁可多写一个 `.as_xxx()` 或 `.to_xxx()`，也不要让同一段源码在不同上下文变成不同类型。**

1.  **废除语境魔法**：字面量不看上下文，不做隐式 ExpectedType 变形推导。彻底消除“代码在不同地方由于语境漂移产生不同类型”的二义性恐慌。
2.  **单向刚性类型流（Rigid Uni-directional Type Flow）**：类型只能“自下而上（From Expr to Expected）”流动，绝对禁止“自上而下”反向操纵。
3.  **大隐于市的人体工学（DX）**：在 95% 的日常纯安全业务编码中，开发者书写最自然的常规双引号 `"..."` 本身就是高安全、零拷贝的 `str`，彻底告别怪异的前缀；而当偶有跨越 FFI 边界或发生所有权交割时，显式的手写转换恰恰是对“物理边界”最清醒的敬畏。

---

## 🏛️ 二、 四大主权角色的物理实相与内存几何

| 主权角色 | 物理实体 / LLVM 内存几何 | 内存所有权（Ownership） | Null-Terminated (\0) | 安全性与生命周期（PAL） |
| :--- | :--- | :--- | :--- | :--- |
| **字面量 `"..."`** | 常驻 `.rodata` 静态段 | **无所有权**（进程同寿 `'static`） | **是**（编译器强制追加） | 100% 静态安全 |
| **`str`** | 胖指针 `{ i8* 首地址, i64 长度 }` | **借用观察轨**（不拥有，不释放） | **否**（切片任意区间） | PAL 铁血镇守其活跃生存期 |
| **`String`** | 堆容器 `{ *#buf, usize, usize }` | **独占拥有轨**（`ScopeUnwinder` 物理 drop） | **是**（堆缓冲区强制封口） | FFI 边界的安全所有权锚点 |
| **`cstr`** | 单指针 `i8*` (零开销 Shape) | **FFI借用轨**（不拥有，无释放） | **100% 静态保证** | 专为跨 C-FFI 边界设计的安全只读单指针 |

## ⚖️ 三、 太极三轨宇宙终极正交转换矩阵

Toka 恪守最严苛的**【开销诚实命名法典】**：
*   **`as_xxx` 家族**：表示**零开销、零分配**的视图借用转换。其借出视图的生命周期将受 PAL 铁血镇守，死死绑定于宿主；
*   **`to_xxx` 家族**：表示**涉及物理拷贝或堆分配**的所有权转换，明确昭示所有权与物理分配的分离。

### 1. 终极正交转换矩阵

| 源类型 $\rightarrow$ 目标类型 | `str` (安全胖借用) | `String` (堆所有权) | `cstr` (\0 借用) | `*char` (FFI 裸指针) |
| :--- | :--- | :--- | :--- | :--- |
| **`"..."` 字面量** | 默认 ResolvedType (0开销) | **`.to_string()`**<br>($O(N)$ 堆分配构造) | **`.as_cstr()`**<br>($O(1)$ 零分配借出) | 自动退化降级<br>(FFI 边界 0摩擦直入) |
| **`str`** | —— | **`.to_string()`**<br>($O(N)$ 堆分配) | **`.to_cstring()`**<br>($O(N)$ 栈/堆拷贝封口) | 坚决禁止隐式！<br>需显式 `.to_cstring().c_str()` |
| **`String`** | **`.as_str()`**<br>($O(1)$ 零分配借出) | —— | **`.as_cstr()`**<br>($O(1)$ 零分配借出) | **`.c_str()`**<br>($O(1)$ 零分配首地址借出) |
| **`cstr`** | **`.as_str()`**<br>($O(1)$ 零分配借出) | **`.to_string()`**<br>($O(N)$ 堆分配) | —— | 自动退化降级<br>(FFI 边界 0摩擦直入) |

---

### 2. 核心流转规则与安全性保障

#### 🚀 规则一：字面量 `"..."` 永远 Resolved 为 `str`（唯一默认）
在不与任何显式形参发生调用碰撞时，常规双引号 `"hello"` 的 ResolvedType 永远固定为 **`str`**。这代表了高安全的第一招牌品味。
*   **日常纯安全逻辑（0前缀摩擦）**：
    ```toka
    auto name = "Toka"  // name 默认推导为 str，受 PAL 安全保护！
    if name.starts_with("To") { ... } // 直接传入常规双引号，0拷贝，0分配，0前缀！
    ```

#### 🚀 规则二：借出视图与堆拷贝命名语义隔离
*   **零开销安全借出**：
    ```toka
    auto view = my_string.as_str()   // 零分配只读胖切片借出
    auto c_view = my_string.as_cstr() // FFI安全只读单指针借出，保证 \0
    ```
*   **物理分配与深拷贝**：
    ```toka
    auto s1 = "hello".to_string()     // 堆分配构造 String
    auto s2 = my_str.to_string()      // 堆分配深拷贝
    ```

#### 🚀 规则三：`str` 到 `cstr` 的“物理拷贝隔离带”
由于普通的动态 `str` 切片（如 substring）并不保证以 `\0` 结尾，为绝对防范跨越 C-FFI 时发生内存越界崩溃：
*   **`str` 坚决、100% 严禁隐式退化为 `cstr` 或裸指针！** 
*   若动态 `str` 确实需要跨越 FFI 边界，开发者必须手写显式转换：
    ```toka
    auto temp_c = my_str.to_cstring() // 显式宣告在栈上（或必要时堆上）拷贝并强制 \0 封口
    ```
    由 `to_cstring()` 返回一个拥有临时安全 `\0` 缓冲区生命周期的中间体，以此安全过界！
