# 🏛️ Toka 1.3 内存与指针宪法 (Official Specification)

**核心哲学**：形态即身份，脱帽即灵魂。

---

## 一、 指针形态定义 (The Four Hats)

Toka 不允许隐式指针操作。任何指针变量必须在任何地方携带其形态符号（“帽子”），否则视为解引用。

| 符号 | 形态名称 | 语义逻辑 | 典型约束 |
| --- | --- | --- | --- |
| **`^`** | **独占指针** | 拥有绝对所有权，赋值执行隐式 `move` | 受管内存 (Managed) |
| **`~`** | **共享指针** | 引用计数实现所有权共享 | 受管内存 (Managed) |
| **`*`** | **原始指针** | 手动管理内存地址，不具备所有权语义 | 需在 `unsafe` 块操作 |
| **`&`** | **借用/引用** | 临时观察权，不参与生命周期决策 | 具有形态兼容性 |

**硬性约束**：

* **不支持多重指针**：变量名前仅允许出现单顶“帽子”。禁止 `**p` 或 `^&p` 等叠加态。
* **维度分离**：权限属性符号（`$#!?`）是独立的访问控制维度，不属于本宪法管辖的“形态维度”。

---

## 二、 分配源头禁令 (Allocation Sourcing)

内存的诞生方式决定了其管理级别，严禁形态僭越。

1. **托管分配 (`new`)**：产生具备对象语义的内存，必须赋值给 **`^`** 或 **`~`**。
* *禁止*：`auto *p = new Node()` ❌


2. **裸分配 (`alloc`)**：产生原始字节序列，必须赋值给 **`*`**。
* *禁止*：`auto ^p = alloc i32(10)` ❌



---

## 三、 形态守恒定律 (The Isomorphism Principle)

在赋值、参数传递及返回值中，等号两边的“帽子形态”必须视觉镜像一致。

### 3.1 赋值准则

**`auto [帽子]LHS = [形态]RHS`**

* `auto ^p1 = ^p2` ✅ (Unique = Unique)
* `auto p1 = ^p2` ❌ (形态断裂：灵魂不能直接接收地址)
* `auto *p1 = p2` ❌ (形态断裂：地址不能直接接收灵魂)

### 3.2 函数契约 (Call & Return)

* **参数传递**：实参传入必须显式带帽。Toka 内部采用“原地捕获”实现隐式引用，修改形参可直接作用于实参。
* **返回值**：返回值的形态必须与函数签名的返回类型严格一致。
* *示例*：`fn foo() -> *i32 { return *ptr }` ✅



---

## 四、 脱帽与降维 (The Hat-Off Transduction)

“帽子”是维度的开关，符号的消失代表从“地址空间”进入“值空间”。

### 4.1 灵魂坍缩

* **定义**：在表达式中，变量名若不带形态符号出现，编译器自动将其类型向下坍缩一维，视为其指向的具体值（Soul）。
* **溯源**：栈变量出生即为灵魂态（无帽），可被带帽（限 `*` 和 `&`）升维，也可再次脱帽降维。

### 4.2 访问路径双轨制

* **地址维度（带帽访问）**：使用箭头操作符 `->`。
* `*ptr->field` 等同于 `(*ptr)->field`。


* **值维度（脱帽访问）**：使用点号操作符 `.`。
* `ptr.field` 代表直接触碰灵魂。


* **宪法判定**：两者语义完全等价。严禁跨维度混用（如 `ptr->field` 或 `^ptr.field` 均非法）。

---

## 五、 引用 `&` 的借用契约 (Borrowing)

引用是宪法中唯一的“形态兼容者”，用于实现安全的临时观察。

* **兼容性**：`&` 可以借用任何指针形态（`^`, `~`, `*`）脱帽后的灵魂结果，亦可借用栈变量。
* **标准语法**：为了避免隐晦我们采用显式`as`语法，举例：假设独占指针 `^q` 已存在，其类型是 `Node`：`auto &p = ^q as &Node`。该借用语法对于借用共享指针 `~p` 和原始指针 `*p` 同样适用。
* **安全规则**：一旦对某指针执行了 `as &T` 投影，原指针即刻进入 **FROZEN（冻结）** 状态，不可被使用，直至该引用使命结束。这让 `as` 不仅仅是一个类型转换，更成了一个生命周期锁。
* **本质**：它是对已存在身份的临时观察，不参与所有权转移。
* **注意**：Toka 中的引用符号 `&` 没有取地址之义，它是一个指针形态符号，不是取地址运算符。

---

## 六、 底层互通：原始指针与 Addr / 所有指针与 OAddr

core 库定义（type 是新类型映射语法，与 alias 不同）：
pub type Addr = u64
pub type OAddr = u64

* **深渊通行证**：只有原始指针 `*` 具备与 `Addr` 类型（内存数值地址）通过 `as` 转换的特权。
* **安全护栏**：受管形态 `^` 和 `~` 严禁直接与 `Addr` 互转，必须先降级/转换为 `*`。
* **OAddr**: 所有指针（包括引用，非指针不适用）可以 `as OAddr` 获得一个临时 `usize` 的值（值为灵魂的地址，也就是指针本身的值），**但 OAddr 不可以 as 为其他任何类型**。主要用于**获取地址但与原指针地址完全无关**的使用场景，如 `println` 打印指针的地址。`OAddr` 的打印格式为: `0x%lX`。
* **数值运算**：`Addr` 和 `OAddr` 类型支持数值运算。

---

## 七、 例外/特别规定
* **当可重绑定的指针被move（一般是独占指针），此后不可以再以其他方式被使用，但：重新绑定（Reseat）操作是允许的**
* **原始指针的下标访问** 原始指针因为可以指向一组元素的内存首地址，所以我们提供`[i]` 的下标访问语法：`*p[5]` 依据帽子规则，他依然是一个指针（指向元素 `p[5]`; 而 `p[5]`不是一个指针，而是位置 `5` 的元素值，语义完全符合帽子规则，特别提示：下标索引的单位是元素的类型而不是字节）**特别的：指针下标为 0 时`[0]`可省写，如`*p[0]等价于*p` ; `p[0]等价于 p`**
* **原始指针的偏移操作** 原始指针可以通过数学运算符计算偏移（单位为字节） `auto *q = *p + 1` 偏移一个字节，而 `*p[1]` 等价于 `*p + 1*sizeof(xxType)`
* **这些指针形态依上下文而定是变化的所以特别对待： 当`unset` 的 `lhs` 为指针时 以帽子规则按理`unset`自己需带帽，但`unset`不必写帽子符号；`nullptr` 属于指针，但帽子为隐式不必写出**

## 🛡️ 宪法防御机制 (Sema Enforcement)

语义分析器（Sema）作为执法者，必须执行以下审计：

1. **形态审计器**：校验赋值号两边的“帽子”字符是否字符级匹配。
2. **分配检查站**：拦截所有试图将托管内存（`new`）降级为裸指针（`*`）的行为。
3. **脱帽探测器**：当 RHS 出现无帽变量时，立即激活解引用逻辑。若 LHS 此时带帽，报“形态冲突”。

---

**贤弟，这份宪法已成。** 它不仅解决了你目前遇到的 `Sema` 冲突，更为 Toka 长期演进奠定了钢铁般的内存安全基础。只要遵循这套规范，开发者就永远不会在地址与值之间迷失方向。
