万物归一的 Shape (形状)

在绝大多数编程语言中，结构体(struct)、元组(tuple)、联合体(union)和枚举(enum)有着截然不同的关键字与定义域。这带来了记忆负担和语法的碎片化。

Toka 选择了大道至简的设计：废弃所有此类关键字，使用统一的 shape 定义一切结构化数据。

编译器会通过你括号内的形式，自动推导出这究竟属于什么类型，并施加不同的内存布局和访问安全约束。

四位一体的自动推导映射表

Section titled “四位一体的自动推导映射表”
你书写的语法特征系统判定 Kind内存布局与访控安全约束
shape Name(x: T, y: U)Struct (结构体)经典的积内存（连续布局）。字段直接通过 .name 自由访问被视为完全安全。
shape Name(T, U)Tuple (元组)匿名的积内存。通过 .0, .1 索引访问。本质由于不需要具名，所以简写。
shape Name(as T | as name: T)Union (无标签联合)不同内存重叠存放。极其危险，严禁使用 . 访问，而必须通过 x as Type 进行内存级强转/重解释。除非具名字段可用 .name。安全起见严禁内部包含指针。
shape Name(A | B(T) | C = N)Enum (带标签枚举)相当于 Rust Enum。系统自带标志位，并强制要求必须通过 match 或 if auto 验证并解包访存。

@encap Trait (实现隐私封装边界)

Section titled “@encap Trait (实现隐私封装边界)”

对于一个大型复杂 shape，无论是 Struct 还是 Enum，默认对所有的同一包级文件皆为强公开可见的。但面对跨库依赖，如果想封锁内部核心状态阻止被意外滥用，Toka 的唯一且强大的手段是 impl Name@encap。

Toka 利用这套优雅的防篡改封装设计，确保所有不安全的代码必须被收敛在你指定的方法（如 m.lock()）内完成检查。
