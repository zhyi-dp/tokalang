简介

Toka 是一门为 性能、安全和简洁 而设计的现代系统级编程语言。它通过独特的 属性令牌系统 (Attribute Token System) 和 指针构词学 (Pointer Morphology)，在高效的底层控制与高层生产力之间架起了一座桥梁。

核心哲学

Section titled “核心哲学”

Toka 建立在几项“铁律”之上，使代码在视觉上具有高度的可预测性和区分度。

1. 戴帽原则 (灵与肉)

Section titled “1. 戴帽原则 (灵与肉)”

与 C 或 Rust 不同，Toka 将变量名视为 Soul (灵/值本身)。直接访问指针变量 p 会自动解引用得到数据。若要操作 Handle (肉/指针地址本身)，你需要通过符号（如 ^, ~, & 或 *）为其“戴上帽子”。

  • p = value (修改 p 指向的数据内容 - 灵魂操作)
  • ^p = ^q (将指针 p 重绑定到一个新的地址 - 肉体操作)

2. 语法令牌属性

Section titled “2. 语法令牌属性”

Toka 使用简洁的令牌来定义可变性和可空性。这些是离散的维度：

  • # (作为名称后缀): 灵魂可变性 (例如：auto x# = 10)
  • ? (作为类型后缀): 实体可空性 (例如：auto x: i32? = none)
  • nul (作为名称前缀): 指针本身可空 (例如：auto nul ^p: User = null)
  • ^#, ~# (作为名称前缀): Handle 可重绑定性 (例如：auto ^#p = new i32(10))

3. 显式所有权

Section titled “3. 显式所有权”

Toka 的指针构词学在类型层面定义了所有权关系：

  • ^T：独占所有权 (RAII)。
  • ~T：共享所有权 (引用计数)。
  • &T：借用访问。

现代特性

Section titled “现代特性”

模式匹配

Section titled “模式匹配”

强大的 match 表达式支持枚举、字符串、整数以及 或模式 (|)。

标准位运算符

Section titled “标准位运算符”

支持标准的 C 风格位逻辑符号：&, |, ^, ~, <<, >>。

AOT 原生编译与零成本抽象

Section titled “AOT 原生编译与零成本抽象”

Toka 是一门纯静态预先编译（AOT）语言，无虚拟机抽象层与庞大的运行时（Runtime）依赖。它依托 LLVM 20 后端，直接生成高效的底层目标平台机器码。

  • 零成本抽象: 语法层面的高级语言特性（如封装、泛型与 RAII）在编译期会被彻底解析并降级，确保运行时无额外的性能开销。
  • 工业级优化: 编译器全面继承并集成 LLVM 的优化管线（支持 -O0 至 -O3 级别），提供卓越的执行效率。
  • 敏捷执行体验: 在保持 AOT 编译极高执行性能的同时，Toka 提供了 toka run 指令与透明的系统级自动链接机制，实现了类似于动态脚本语言的敏捷开发工作流。
